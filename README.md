Проект на основі Spring Security, контролерів, сервісів та html-шаблонів (Thymeleaf). 

Ідентифікація користувача виконується засобами Spring Security.
Введено поділ користувачів за ролями ADMIN,USER,MODERATOR (cторінка модератора поки що відсутня).
Обмежено доступ до певних URL сайту залежно від того чи пройшов аутентифікацію користувач та чи має він відповідну роль (файл SecurityConfig).

Містить: 
- форму реєстрації (доступна всім), 
- форму авторизації користувачів (Spring Security перенаправляє на цей URL при спробі доступу до сторінок, що потребуюють авторизації), 
- особисту сторінку користувача (доступна всім авторизованим користувачам),
- сторінку зі списком користувачів (доступна користувачу з роллю ADMIN).
